## 字段说明

| 字段 | 必填 | 类型 | 说明 |
|------|------|------|------|
| id | ✅ | string | 唯一标识：`{学科}_{年份}_{序号}` |
| subject | ✅ | string | 学科缩写（对应 subjects.json） |
| category | ✅ | string | 学科中文名 |
| subcategory | ⬜ | string | 子分类 |
| type | ✅ | string | `single` / `multiple` / `uncertain` |
| stem | ✅ | string | 题干 |
| options | ✅ | string[] | 选项（含ABCD前缀） |
| answer | ✅ | string / string[] | 单选："B"；多选：["A","C"] |
| analysis | ✅ | string | 详细解析 |
| rule | ⬜ | string | 核心规则一句话 |
| lawRef | ⬜ | string | 法条依据 |
| trap | ⬜ | string | 陷阱说明 |
| difficulty | ⬜ | number | 1-5 难度 |
| tags | ⬜ | string[] | 标签 |
| year | ⬜ | number | 年份 |
| source | ⬜ | string | 来源：真题/模拟题/改编 |

## 导入方式

1. **设置页导入**：设置 → 题库管理 → 导入题库 → 选择 JSON 文件
2. **直接放入 data/questions/**：新文件会在 app.js 的 fetch 列表中自动加载（需手动注册）

## 质量标准

- 每题必须有 `analysis`（解析是学习价值的核心）
- `answer` 必须对应 options 中存在的选项
- `id` 全局唯一，推荐 `{学科}_{年份}_{3位序号}` 格式
- 难度 1-2 为基础题，3 为中等，4-5 为拔高
